# Commit, Push & PR

## Порядок:
1. `git status` — проверь что коммитишь
2. `git diff --staged` — просмотри изменения
3. Коммит с осмысленным сообщением (не "fix" или "update")
4. Push на фича-ветку
5. Создай PR с описанием: что изменилось, кого затрагивает, как откатить

## Формат коммит-сообщения:
```
<тип>: <краткое описание>

<детали, если нужно>
```

Типы: feat, fix, refactor, docs, test, chore
